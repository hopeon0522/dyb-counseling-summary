# 상담요약 PWA

iPhone Safari에서 접속한 뒤 홈 화면에 추가해서 사용하는 상담요약 웹앱입니다. Xcode로 설치하는 앱이 아니므로 무료 Apple Developer 계정의 7일 갱신 제한이 없습니다.

## 프로젝트 구조

```text
web/
  index.html
  styles.css
  app.js
  manifest.json
  service-worker.js
  icon.svg
README.md
```

이 프로젝트의 현재 개발 대상은 `web/` PWA입니다. Xcode/iOS 네이티브 앱은 사용하지 않습니다.

## iPhone Safari에서 접속하는 방법

정적 파일 서버를 실행합니다.

```sh
cd web
python3 -m http.server 8080
```

Mac과 iPhone을 같은 Wi-Fi에 연결한 뒤 Mac IP를 확인합니다.

```sh
ipconfig getifaddr en0
```

iPhone Safari에서 아래 주소로 접속합니다.

```text
http://Mac-IP:8080
```

로컬 HTTP 주소는 개발 테스트용입니다. 서비스워커 캐시와 설치형 PWA 동작은 HTTPS 배포 환경에서 가장 안정적입니다.

## 홈 화면에 추가하는 방법

1. iPhone Safari에서 웹앱 주소를 엽니다.
2. 공유 버튼을 누릅니다.
3. `홈 화면에 추가`를 선택합니다.
4. 홈 화면에 생긴 `상담요약` 아이콘으로 실행합니다.

디자인 파일이 갱신되지 않으면 홈 화면 앱을 완전히 종료한 뒤 다시 열거나, Safari에서 새로고침하세요. 그래도 이전 화면이 남으면 홈 화면 아이콘을 삭제하고 다시 추가하면 됩니다.

## 화면 모드

설정의 `화면 모드`에서 선택할 수 있습니다.

- `시스템 설정`: iPhone의 라이트/다크 모드를 따라갑니다.
- `라이트 모드`: 밝은 배경과 코랄 포인트 색상을 사용합니다.
- `다크 모드`: 어두운 배경과 코랄 포인트 색상을 사용합니다.

## Gemini API 키 발급 및 입력 방법

1. Google AI Studio에서 Gemini API 키를 발급합니다.
2. 웹앱의 설정 화면을 엽니다.
3. `분석 모드`를 `Gemini 모드`로 바꿉니다.
4. `Gemini API 키`에 발급받은 키를 입력합니다.
5. `설정 저장`을 누릅니다.

기본 모델은 Flash 계열인 `gemini-2.5-flash`입니다. 설정 화면에는 사용 중인 버전만 표시되고, 사용자가 직접 모델명을 수정하지 않습니다.

## iPhone에서 단독으로 실행하는 방법

Mac의 로컬 웹서버 없이 iPhone에서 독립적으로 쓰려면 `web/` 폴더를 HTTPS 정적 호스팅에 올려야 합니다.

권장 방식:

- GitHub Pages
- Netlify
- Cloudflare Pages
- Vercel 정적 배포

배포 후에는 iPhone Safari에서 배포 주소를 열고 `홈 화면에 추가`를 하면 됩니다. 이후에는 Mac이 꺼져 있어도 웹앱 화면은 열립니다. Gemini 모드는 iPhone에서 Gemini API로 직접 요청을 보내므로 인터넷 연결과 유효한 API 키가 필요합니다.

주의: 브라우저에서 Gemini API를 직접 호출하는 방식은 API 키가 iPhone 브라우저 저장소에 남습니다. 혼자 쓰는 개인용으로는 간단하지만, 다른 사람에게 배포하거나 키 보안을 더 강하게 하려면 백엔드 프록시 방식이 더 안전합니다.

## 무료 티어 사용 시 주의사항

- 같은 상담 내용으로 요약 버튼을 반복해서 눌러도 같은 요청은 다시 보내지 않도록 막았습니다.
- 요약 중에는 버튼이 비활성화됩니다.
- Gemini 요약 중에는 준비, 전송, 응답 대기, 후처리 단계를 기준으로 예상 진행률을 표시합니다. Gemini API가 실제 토큰 처리율을 제공하는 것은 아니므로 정확한 서버 내부 진행률은 아닙니다.
- 요약 실패 시 토큰/입력 길이 문제, 무료 티어 한도, API 키 권한, 네트워크/CORS 문제를 가능한 한 구분해 안내합니다.
- 참고 예시는 최근 예시 3개까지만 프롬프트에 포함합니다.
- 각 참고 예시는 앞 1,200자까지만 포함합니다.
- 원하는 요약 분량은 저장된 참고 예시 평균 글자 수를 기준으로 계산합니다.
- 기본 범위는 예시 평균 기준 하한 -20% / 상한 +20%이며, 설정에서 10% 단위로 바꿀 수 있습니다.
- 긴 상담 원문은 앞부분, 중간, 끝부분 중심으로 자동 축약해 한 번만 요청합니다.
- 무료 한도 초과 또는 분당 요청 제한이 발생하면 잠시 기다렸다가 다시 시도하세요.

## 참고 예시로 페르소나 만들기

상단의 `샘플` 탭에서 상담요약 예시를 저장할 수 있습니다.

- `텍스트 파일로 예시 추가`: txt 파일을 선택해 예시로 저장합니다.
- `직접 예시 추가`: 상담요약 예시를 직접 입력합니다.
- 저장된 예시는 문체, 표현 방식, 요약 분량, 중요 포인트 선택 기준을 맞추는 페르소나로 사용됩니다.
- 저장된 예시의 평균 글자 수가 요약 분량 기준이 됩니다.
- 샘플 예시는 브라우저 localStorage에 저장되므로 앱을 껐다 켜도 유지됩니다. Safari 웹 사이트 데이터를 지우거나 홈 화면 앱 데이터를 삭제하면 함께 지워질 수 있습니다.

## 요약 결과 형식

요약 결과는 별도 화면으로 이동하지 않고 Home의 `요약하기` 버튼 아래에 표시됩니다.

출력 형식:

```text
[학습습관 점검]
- 상담 내용 메모체로 정리하였음
- 학부모 상담 내역에 맞게 설명드림, 논의드림, 언급드림 형태로 작성함
- 문장 끝 마침표 없이 작성함
```

결과 아래에서 바로 복사하거나 이메일 작성으로 이동할 수 있습니다.

## API 키 보안 안내

이 PWA는 사용자가 직접 입력한 Gemini API 키를 브라우저 `localStorage`에 저장합니다. 키를 코드에 직접 넣지 마세요.

공용 기기에서는 API 키를 저장하지 않는 것이 좋습니다. 더 강한 보안이 필요하면 별도 백엔드 프록시를 두고 API 키를 서버 환경변수로 관리하는 방식이 안전합니다.

## mailto 방식 이메일 전송의 한계

이메일 보내기는 `mailto:` 링크를 사용합니다.

- 실제 발송은 사용자의 Mail 또는 Gmail 앱에서 진행됩니다.
- 메일 앱이 설정되어 있지 않으면 작성 화면이 열리지 않을 수 있습니다.
- 본문이 너무 길면 메일 앱이나 브라우저가 일부 내용을 잘라낼 수 있습니다.
- 자동 발송이 필요한 경우에는 별도 이메일 발송 서버가 필요합니다.

## 긴 상담 파일 처리 방식

상담 원문이 너무 길면 Gemini 무료 티어 사용량을 줄이기 위해 앱이 자동으로 다음 부분만 묶어 요청합니다.

- 앞부분
- 중간 구간
- 끝부분

이 방식은 통화의 시작 맥락, 중간 상담 흐름, 마지막 조치 내용을 최대한 보존하기 위한 절충안입니다. 아주 긴 상담에서 세부 내용이 빠질 수 있으므로 중요한 대목은 직접 붙여넣기 영역에서 정리해 넣는 것이 좋습니다.

## 개발 모드

설정의 `분석 모드`를 `Mock 모드`로 두면 Gemini API 키 없이도 화면 이동, 요약 결과 표시, 복사, 이메일 작성 흐름을 테스트할 수 있습니다. 실제 API 비용이나 무료 티어 사용량이 발생하지 않습니다.
